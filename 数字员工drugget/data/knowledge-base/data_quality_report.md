# 数据清洗与质量报告

## 数据范围

- 店铺档案：10,507 行，来源为 `网络店铺档案明细表_2026.xlsx`。
- 趣维历史：17,822 行识别为正式药品记录。
- 安托历史：168,797 行识别为正式药品记录。
- 控价规则：31 条；同一商品存在多个规格规则时保留为多条。

## 清洗动作

1. 重置 Excel 错误的工作表维度后读取实际数据，不改写原始文件。
2. 统一平台编码、商品名、规格、包装数量和最小单位。
3. 从 URL 或商品 ID 提取商品身份；无法提取的记录进入质量问题清单。
4. 按平台+店铺名匹配当前店铺档案，无法匹配的记录不进入责任核心。
5. 完全重复事实移除；同业务键但价格不同的记录保留并单独统计。
6. 以控价标准表计算标准化单件价状态，同时保留历史来源侧标记。

## 发现与风险

|问题|数量|严重度|影响|
|---|---:|---|---|
|未识别药品|354|中/高|无法可靠关联药品知识|
|店铺未匹配|6,075|中|不能自动路由责任人|
|缺少商品身份|7,975|高|不能生成稳定复查任务|
|单盒价公式不一致|164|高|金额计算存在可信性风险|
|完全重复移除|1,090|中|避免重复计数|
|同业务键重复额外行|29,082|高|需结合采集轮次/价格判断是否为重复抓取|

详细行级记录见 `data_quality_issues.csv`，生成摘要和源文件哈希见 `manifest.json`。
